- Implemented agent bus as a hybrid model: in-memory channel routing for local delivery plus Wolverine `IMessageBus.PublishAsync` for cross-service propagation.
- Backpressure policy is enforced at enqueue-time with a hard cap of 100 pending messages; when saturated, the bus drops the oldest low/normal-priority item first, and rejects newly arriving low/normal messages if no droppable item exists.
- Local routing supports three modes without direct agent-to-agent coupling: direct (`TargetAgentId`), topic (`Topic`), and broadcast (fallback when target/topic absent).
- Message audit persistence is centralized through `AgentMessagePersistence`, using scoped `MimirDbContext` resolution from singleton bus context and duplicate-write protection by `MessageId`.
- Audit payloads are sanitized before persistence via JSON-node redaction of sensitive keys (email/ip/phone/ssn/token/password) to avoid storing PII in agent message logs.
- Added `AgentMessage` entity + EF configuration + `DbSet` to ensure agent communication is queryable and auditable in the relational store.
- Added Wolverine bridge handler (`WolverineAgentMessageHandler`) that routes externally received agent messages into the local in-memory bus path.
- Added focused infrastructure tests for direct routing, broadcast routing, topic routing, incoming-route bridging behavior, and queue-cap backpressure behavior using NSubstitute + Shouldly.
- Implemented a background task executor with `Channel<BackgroundTaskItem>` queue + `ConcurrentDictionary` task state, and moved queue processing into a hosted `BackgroundTaskProcessor` guarded by `SemaphoreSlim` for configurable concurrency.
- Added task lifecycle persistence via `BackgroundTask` entity and EF state store, with transitions for queued/running/completed/failed/cancelled and a migration introducing `background_tasks` table with status/submitted and unique task-id indexes.
- Used scoped resolution through `BackgroundTaskRuntime` from singleton executor/processor to safely invoke scoped orchestrator and EF state persistence from background workers.
- Added 1-hour result caching with `IMemoryCache` and timeout handling via linked `CancellationTokenSource` + per-task timeout fallback from options.
- Added focused `BackgroundTaskExecutorTests` using NSubstitute + Shouldly to validate submit/list, completion and cache behavior, cancellation flow, and timeout path.

- KnowHub bridge integration in Mimir compiles when adding direct ProjectReferences from Mimir.Infrastructure to nem.Cognitive.Abstractions/Distillation/GraphRag/Graph and wiring AddKnowHubIntegration into infrastructure DI.
- Bridge degradation pattern implemented as per-call try/catch with warning logging + empty fallback payloads; availability tracked with a probe interval and IEmbeddingService.GetDimensionsAsync health probe.
- For this workspace, Roslyn LSP may report stale unresolved external-reference diagnostics in test files even when dotnet build + targeted tests pass; treat build/test as source of truth for cross-repo ProjectReference validation.
- Agent memory context assembly in Mimir is cleanly modeled as an application service that returns a transport-friendly `MemoryContext` record (`SemanticFacts`, `EpisodicSummaries`, `WorkingMessages`, `TotalTokenEstimate`) so orchestrators can inject pre-assembled context without coupling to memory store contracts.
- Parallel memory retrieval with `Task.WhenAll` plus per-store `Task.WhenAny` timeout fallback provides low-latency context assembly: each store independently degrades to empty results on timeout/failure while preserving successful store outputs.
- Agent-type access control is easiest to keep deterministic by resolving a normalized access profile (`explore`/`general`/`execute`/default) that controls both store enablement and budget multipliers before launching retrieval tasks.
- Budget enforcement is most reliable when done twice: first allocate per-store token budget percentages from the global cap, then truncate formatted output lines by estimated token cost so returned context cannot exceed configured allocation.
- Memory consolidation orchestration in Application should isolate stages with independent try/catch blocks (Working→Episodic and Episodic→Semantic) so episodic failures do not block semantic attempts, and should degrade gracefully by short-circuiting semantic distillation when `IKnowHubBridge.IsAvailable` is false.
- Prompt-level compression can be implemented as a standalone application service that complements (not replaces) `IContextOptimizer`: keep context-window management in `ContextOptimizerService` and handle system/few-shot/tool prompt shaping separately.
- Preserving agent behavior during compression is safest when user blocks are passed through verbatim, safety/system directives are explicitly retained, and only non-user prompt scaffolding is compacted/pruned.
- Relevance-based few-shot/tool filtering can remain deterministic with token-overlap scoring against normalized query terms plus a hard token budget for tool blocks.
- Current `Mimir.Application.DependencyInjection` exposes no configuration parameter in `AddApplicationServices`; for parity with existing patterns, prompt-compression options registration was added as `Options.Create(new PromptCompressionOptions())` singleton and service scoped registration.
